# Design Principles

1. **MVP honesty over overclaiming**
   - Clearly state what is implemented vs conceptual.

2. **On-chain publication integrity first**
   - Establish verifiable global state before advanced anti-cheat.

3. **Keep global state minimal**
   - Best-score-per-player over full action/event history.

4. **Deterministic ranking**
   - Same input state should always produce same leaderboard order.

5. **Roadmap-ready interfaces**
   - Reserve fields/hooks for future verification without destabilizing MVP.

6. **Mobile/hackathon feasibility**
   - Prefer lower-complexity, demonstrable architecture under time constraints.
